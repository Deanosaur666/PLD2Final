# Final Project

Determine the frequencies of title-cased words (proper nouns and sentence starters) in a large English text file, such as a book. Exclude acronyms from consideration. Use a manager-worker architecture.

Submit the technical report and the MPI code as a zipped **Final Project** folder to Canvas. Follow the guidelines from the _How to Write the Technical Report_ document. Specifically, the report should include:

1. Problem description.
2. Programming environment: operating system, IDE, processor specifications including the number of cores.
3. Design (the four steps of Foster’s methodology).
4. Benchmarking (with printing disabled) for three different values of text size and 1, …, 10 processors. For each number of processors and each text size, determine the average execution time after five runs. Plot the results using a line graph.
5. Discussion.

**Grading:**

1. Code (30%)
2. Report (70%), as follows:
	1. Problem description and programming environment included in the report (5%)
	2. Design included in the report (20%)
	3. Benchmarking included in the report (20%)
	4. Discussion included in the report (5%)
	5. Writing style of the report (20%). Here are some guidelines:
		1. Tables, graphs, and figures should be labeled properly and referenced in the text
		2. Avoid writing the report as a list of bulleted paragraphs and focus on structuring your content into well-formed sections with clear, flowing prose
		3. Present findings with analysis and interpretation
